# 압구정곱창 메뉴 분석 및 정규화

## 분석 상태
- **현재 진행률**: 0% (원본 입력 대기중)
- **원본 파일 위치**: `_raw/원본_입력폴더/`
- **마지막 업데이트**: 2026-04-15

## 단계별 진행

### 1️⃣ 원본 입력 단계
원본 파일을 `_raw/원본_입력폴더/` 에 넣으세요.

**지원 형식**:
- Excel / CSV
- 이미지 (메뉴판 사진)
- PDF (메뉴 문서)
- 텍스트 (메뉴 리스트)

### 2️⃣ 분석 단계
원본 데이터를 다음 기준으로 분류:

| 항목 | 기준 | 예시 |
|------|------|------|
| **카테고리** | 메뉴 종류 | 곱창류, 소곱창류 |
| **메뉴명** | 정확한 이름 | 대곱창 (양념/구성 제외) |
| **가격** | 기본가격 | 15,000원 → 15000 |
| **설명** | 간단한 설명 | "신선한 소의 대곱창" |
| **옵션** | 선택사항 | 크기, 매운맛 등 |

### 3️⃣ 정규화 단계
DB 테이블에 맞춰 데이터 입력:

```
menu_categories.json  → 카테고리 분류
menu_items.json       → 메뉴 항목
menu_options.json     → 추가 옵션
```

### 4️⃣ 메뉴얼 생성 단계
정규화된 데이터로 최종 메뉴얼 문서 생성

|------|------|

| **메뉴 항목** | 0개 (대기중) |
| **옵션** | 0개 (대기중) |

## 정규화 규칙

### 가격 처리
```
입력: "15,000원", "대곱창 15000"
출력: 15000 (숫자만)
```

### 메뉴명 정규화
```
입력: "대곱창 (양념/구성)"
출력: "대곱창" (카테고리와 옵션 분리)
```

### 카테고리 표준화
```
곱창류, 대곱창류, 소곱창류, 기타곱창 → 표준 카테고리로 통일
```

### 설명 길이
```
최대 100자 이내 (간결성)
```
